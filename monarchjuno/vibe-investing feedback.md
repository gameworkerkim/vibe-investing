## 1. 레포 개요 (Snapshot)

`vibe-investing`은 Anthropic의 **Claude Skills 스펙(`SKILL.md` 기반)**에 맞춰 투자·금융 워크플로우를 모듈화하려는 초기 단계의 스킬 라이브러리다. 저자(monarchjuno)는 "vibe coding"의 연장선으로 "vibe investing" 컨셉을 들고 나왔으며, README의 핵심 명제는 다음과 같다.

> "Harness engineering is not only for vibe coding. It matters for vibe investing too."

즉, **LLM에게 투자 분석을 맡길 때도 코딩과 마찬가지로 "스킬 단위 모듈화"라는 규율(harness)이 필요하다**는 주장이다.

### 1.1 정량 지표
| 항목 | 값 |
|---|---|
| 커밋 수 | 2 (사실상 초기 스캐폴딩 1~2회) |
| 스타 | 2 |
| 활성 스킬 | 1개 (`fundamental-analysis/company-analysis`) |
| 카테고리 폴더 | 5개 (대부분 빈 껍데기) |
| 라이선스 | 명시 없음 ⚠️ |
| README/AGENTS.md | 존재 (구조·거버넌스 규정만) |

### 1.2 설계 철학 (AGENTS.md 요약)
- **Capability-first 카테고리화**: 자산군(주식/크립토/채권)이 아니라 분석 능력(fundamental, quantitative, portfolio…) 기준으로 분류 → 하나의 스킬이 여러 자산군을 커버.
- **얇은 SKILL.md + 깊은 references/scripts/assets 구조** → 스킬 본문은 짧고 실행 지향, 디테일은 분리.
- **호스트 런타임 비종속**: Claude Code뿐 아니라 일반 Skill Spec을 따르려는 의도.
- **소규모 점진적 수정**: 광범위 리팩터 대신 작은 PR 기반.

### 1.3 카테고리 5개와 로드맵
| 카테고리 | 현재 상태 | 권장 다음 스킬 |
|---|---|---|
| `fundamental-analysis` | 1개 활성 (company-analysis) | `crypto-analysis` |
| `market-analysis` | 비어있음 | `macro-analysis` |
| `quantitative-analysis` | 비어있음 | `factor-analysis` |
| `portfolio` | 비어있음 | `portfolio-construction` |
| `output-formats` | 비어있음 | `financial-report` |

## 2. 유용성 평가 (Utility)

### 2.1 강점
1. **컨셉의 시의성** — "Skills"는 Anthropic이 2025년 후반부터 본격 푸시한 신규 확장 메커니즘이고, 금융 도메인의 SKILL.md 자산은 아직 시장이 비어있다. 진입 시점 자체는 합리적이다.
2. **구조적 정합성** — Capability-first 카테고리는 실제로 좋은 선택이다. "주식 DCF 스킬"과 "크립토 토큰 가치평가 스킬"을 분리하면 중복이 생기지만, "valuation" 스킬 하나로 자산군별 분기 처리가 가능하다.
3. **거버넌스 명문화** — `AGENTS.md`가 폴더 단위로 존재하는 패턴은 다인 협업·LLM 에이전트 협업 모두에 유리하다. 디스커플 잘 잡혀있다.
4. **얇은 SKILL.md 원칙** — 컨텍스트 토큰을 아끼는 실전적 설계. Claude Skills의 "progressive disclosure" 원칙과 일치.
5. **출력 포맷 분리** — `output-formats` 카테고리를 따로 둔 점, 그리고 docx/pdf/xlsx 같은 파일 변환은 외부 스킬로 위임한다고 명시한 점은 OS급 책임 분리 사고방식.

### 2.2 약점 (현재 시점)
1. **사실상 빈 레포** — 활성 스킬 1개. README/AGENTS.md만 보면 청사진은 그럴듯하지만, 검증할 코드도 레퍼런스도 거의 없다. 현재로선 "투자 스킬 라이브러리"가 아니라 "투자 스킬 라이브러리를 만들겠다는 선언문"이다.
2. **검증된 유스케이스 부재** — 백테스트 결과, 실제 투자 의사결정 사례, 사용자 피드백 등 효용을 입증할 어떤 증거도 없다.
3. **라이선스 누락** — 상업 활용 가능 여부 불명. 기업 도입은 이 시점에선 어렵다.
4. **경쟁 레포가 이미 존재** — `yennanliu/InvestSkill`, `openclaw/skills`, `anthropic/claude-cookbooks`의 financial-models 스킬, `K-Dense-AI/claude-scientific-skills` 등은 이미 수십~수천 개 스킬을 보유하고 활성 사용자를 보유. `vibe-investing`은 차별화 포인트가 아직 보이지 않는다.
5. **"harness engineering" 비유의 한계** — 코드 버그는 실행 즉시 드러나지만, 잘못된 투자 분석은 수개월~수년 후에 손실로 드러난다. 같은 "harness" 개념이라도 검증 메커니즘이 근본적으로 다르며, 레포는 이 차이에 대한 인식이 보이지 않는다.

## 4. 전통 퀀트 트레이딩 관점의 리스크 (Quant Risk Audit)

이 섹션이 가장 중요하다. **"vibe investing"이라는 명명 자체가 퀀트 진영에서는 거의 적색 신호**다. 진지한 퀀트 관점에서 본 구조적 리스크를 정리한다.

### 4.1 방법론적 리스크 (Methodological)

**R1. 백테스트 프레임워크 부재**
- 레포에 backtesting, walk-forward analysis, out-of-sample validation에 대한 어떤 언급도 없다.
- 퀀트 관점에서 검증되지 않은 시그널은 시그널이 아니라 **사후 합리화(narrative)**다.
- 현재 카테고리 `quantitative-analysis`는 "factor, screening, backtesting, signal research"를 포괄한다고 README에서 약속하지만, 백테스트로 검증 정보가 부재

**R2. 룩어헤드 바이어스(Look-ahead Bias)**
- LLM 기반 분석의 근본 문제: 모델 학습 데이터에 미래 정보가 포함되어 있다. 예) "2024년 저점에서 NVDA를 매수했어야 한다"는 분석은 LLM에게는 매우 쉽지만, 2024년 시점 정보만으로 동일 결론을 내는지는 별개 문제.
- SKILL.md 어디에도 시점-제약(point-in-time) 데이터 처리 가이드가 보이지 않는다.

**R3. 생존 편향(Survivorship Bias)**
- 현재 상장사 기반 분석은 상장폐지된 기업들을 자동 누락한다.
- 이 문제에 대한 인지·대응 가이드가 SKILL.md에 명시되어야 하는데 부재.

**R4. 과적합(Overfitting) — "내러티브 과적합"**
- 전통 퀀트 과적합은 파라미터 튜닝에서 발생.
- LLM 기반 분석에서 과적합은 **"그럴듯한 스토리 생성 능력"**으로 발생. 어떤 종목/사건에 대해서도 사후적으로 합리적 설명을 만들어낼 수 있다.
- 이 위험을 통제할 메커니즘(예: 사전등록된 가설, 반증 가능성 검증)이 레포에 없다.

### 4.2 실행 리스크 (Execution)

**R5. 거래 비용·슬리피지 무시**
- 모든 알파(α)는 거래 비용을 차감해야 의미가 있다. 펀더멘털 분석 스킬이 매수/매도 시그널을 도출해도, 슬리피지/수수료/세금 모델이 없으면 paper trading 수준.
- 특히 한국 시장(거래세 0.18% + 양도세) / 크립토(거래소별 메이커-테이커 수수료 차이)에서 이 누락은 치명적.

**R6. 시장 미시구조(Market Microstructure) 무시**
- 호가 두께, 일중 변동성, 유동성 시간대 분포 등은 레포의 어떤 카테고리에도 포함 안 됨.
- 크립토 알트코인이나 한국 코스닥 소형주에 적용 시 모델과 현실의 괴리 거대.

**R7. 체결 모델·포트폴리오 회전율**
- `portfolio` 카테고리는 "allocation, sizing, rebalancing, risk budgeting"을 약속. 그러나 회전율 제약, 거래 일자 제약, 단일 종목 비중 제약 등 실무 제약 가이드는 0.

### 4.3 통계·확률적 리스크 (Statistical)

**R8. 샘플 사이즈 / Multiple Hypothesis Testing**
- AI 기반 종목 스크리닝은 본질적으로 수천 가설을 동시 검정하는 행위.
- Bonferroni / FDR 보정에 대한 가이드 없이 "AI가 추천한 종목"을 따르는 것은 **p-hacking의 자동화**.

**R9. Regime Change 미인식**
- 모든 백테스트는 그 기간의 시장 레짐(저금리/고금리, 저변동성/고변동성)에 의존.
- 레짐 식별·전환 감지 스킬은 `market-analysis` 카테고리에 들어갈 후보지만 구현 0.

**R10. 신뢰도 보정(Confidence Calibration) 결여**
- LLM은 잘못된 답에도 자신감 있게 응답한다. 투자 의사결정에서 이는 직접적 손실 원인.
- 일부 경쟁 스킬(예: openclaw의 fundamental-stock-analysis)은 confidence level 명시를 강제하지만, `vibe-investing` SKILL.md에는 해당 패턴 없음.

### 4.4 거버넌스·규제 리스크 (Governance)

**R11. 투자자문 규제**
- 한국 자본시장법상 투자자문업·유사투자자문업 규제 적용 가능성. LLM 산출물을 그대로 외부 배포하면 회색지대.
- 레포에 disclaimer 가이드, "교육·연구용" 명시 패턴 등 부재.

**R12. 데이터 출처·라이선스**
- 펀더멘털 분석 스킬은 어떤 데이터 소스를 가정하는가? FactSet/Bloomberg/SimplyWall.st? 한국이면 KRX/금감원 DART?
- 데이터 라이선스 위반 시 상업화 봉쇄.

**R13. 책임 할당(Accountability)**
- "vibe investing"이라는 명칭 자체가 **의사결정의 블랙박스화**를 정상화한다. Collin Wallace의 원문도 "투자자가 왜 그 결정을 내렸는지 본인도 모른다"는 점을 핵심 정의로 삼는다.
- 손실 발생 시 누가 책임지는가에 대한 워크플로우가 없으면 펀드 운용·고객 자문 어디에도 적용 불가.

### 4.5 행동·심리적 리스크 (Behavioral)

**R14. 자동화 편향(Automation Bias)**
- 사용자는 LLM 산출물을 비판 없이 수용하는 경향이 있다. "vibe"라는 표현은 이 경향을 더욱 조장.

**R15. 강세장 함정**
- Wallace 원문 인용: vibe investing은 **시장이 오를 때만 즐겁다**. 진짜 시험은 코드가 깨질 때 / 투자가 흔들릴 때 자신의 결정을 변호할 수 있는가다.
- 이 레포는 강세장 보조 도구로는 매력적이지만, 약세장·드로다운 국면 대응 스킬이 결여.
